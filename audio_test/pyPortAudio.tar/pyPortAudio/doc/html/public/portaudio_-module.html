Read(...)

pa_Read Arguments:
  • stream - stream handle
  • nframes - number of frames to read and return
Returns:
  • sequence of frames. Each 'frame' is a tuple
Notes:
  • If there aren't enough audio frames available to satisfy this request, this function will block until enough frames come available
  • This is one painfully slow function. There's a huge overhead in converting raw C buffers into python sequences. If you need speed, and don't need to be able to process the received data, use ReadRaw() instead

ReadRaw(...)

pa_ReadRaw Arguments:
  • stream - stream handle
  • nframes - number of frames to read and return
Returns:
  • raw frame data, returned as a string. Not very useful if you want to do any processing, but MUCH faster than Read()!
Note:
  • If there aren't enough audio frames available to satisfy this request, this function will block until enough frames come available

ReadWavFile(...)

pa_ReadWavFile

Read a WAV file into a sequence of frames

Arguments:
  • file - path of WAV file to read
  • channels - number of channels requested for output
Returns:
  • wav data - tuple (list_of_frames_tuples, samplerate, channels)
Note:
  • to play this WAV data, use Write, not WriteRaw

ReadWavFileRaw(...)

pa_ReadWavFileRaw

Read a WAV file into a raw buffer string

Arguments:
  • file - path of WAV file to read
Returns:
  • tuple:
    • raw data as a string
    • number of frames
    • sample rate
    • channels
Note:
  • to play this WAV data, use WriteRaw(), not Write()

Write(...)

pa_Write

Write data to audio stream

Arguments:
  • stream - handle of stream to write to
  • frames - sequence of frames - each frame is a 1 or 2-element tuple, according to whether the stream is mono or stereo
Returns
  • None
Notes:
  • This routine will block untill all the data has been written

WriteRaw(...)

pa_WriteRaw

Write raw data to audio stream

Arguments:
  • stream - handle of stream to write to
  • buf - string of raw data to write
Returns
  • None
Notes:
  • This routine will block untill all the data has been written
  • Faster than Write(), at a price of not being able to process the data
